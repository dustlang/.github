# Security Policy

## Reporting a vulnerability
Please report security issues **privately**.

Preferred: GitHub Security Advisories (Security tab → Report a vulnerability).

Include:
- Impact assessment
- Reproduction steps / proof-of-concept (if safe)
- Affected versions / commits
- Suggested mitigation (if known)

Do **not** open public issues for security vulnerabilities.

## Supported versions
Active development branches are supported. Older tags may not receive fixes.

## Disclosure
We coordinate responsible disclosure and will credit reporters where appropriate.
